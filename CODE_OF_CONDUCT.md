# Code of conduct

Be useful, be honest, don't be a jerk. Two extra clauses specific to
this project:

- **Political neutrality.** The dashboard reports public data; it
  does not campaign. Comments, PRs, and review discussions that try
  to tilt the platform toward a party, candidate, or policy position
  will be moderated the same as any other off-scope change.
- **No targeting of individuals.** Constituents are not data
  subjects here. Elected officials in their public capacity (voting
  records, expenses, sitting hours) are; their private lives aren't.

## Expected behaviour

- **Useful** — reports have reproduction steps (council, metric,
  observed value, expected value).
- **Honest** — if you're unsure which licence an upstream uses, say
  so. Don't guess.
- **Respect the audience** — end users include citizens who don't
  read dashboards for a living. Error messages and tooltips should
  be plain English.
- **Good-faith reading** — ask before assuming hostility.

## Not tolerated

- **Personal attacks** on the maintainer, contributors, or anyone
  CC'd in an issue or PR.
- **Harassment of any kind.** Zero tolerance.
- **Campaigning via the issue tracker or PR descriptions.** Keep it
  on merits of the data and the code.
- **Doxxing or harassment of named officials.** Link to public
  records only; do not aggregate them into a profile designed to
  intimidate.
- **Commercial spam.**
- **Bad-faith reports.**

## Scope

Issues, PRs, discussions, code review comments, and public
communication about the project on external platforms.

## Reporting

Abuse routes through the same private channel as security:

<https://github.com/Pawansingh3889/forthepeople-uk/security/advisories/new>

Subject line: "Code of conduct". Acknowledgement within 7 days.

## Enforcement

1. **Private warning.**
2. **Temporary block** — days to weeks.
3. **Permanent block** — for harassment that resumes, or for
   threats / doxxing / campaigning in bad faith.
